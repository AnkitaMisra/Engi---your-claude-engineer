# /explain — explain a file or function
# Usage: /explain [file or function name]

Read the specified file or function and explain:
1. What it does in plain English
2. Why it exists — the business/technical problem it solves
3. How it connects to the rest of the system
4. Any non-obvious decisions or patterns used

Audience: a developer seeing this code for the first time.
Be concise — no longer than needed.
