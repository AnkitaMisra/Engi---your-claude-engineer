# /docs — regenerate the docs/README.md index
# Run this any time to sync the index with what's actually in docs/
# Usage: /docs

Scan the docs/ folder and regenerate docs/README.md from scratch.

## Step 1 — Scan docs/ folder

Find all files in docs/:
- docs/architecture/*.md  → architecture docs
- docs/plans/*.md         → technical plans (exclude timestamped copies)
- docs/reviews/*.md       → any saved reviews
- Any other .md files in docs/

A timestamped copy has a date pattern in the name: YYYY-MM-DD.
List them separately under "Archive" — don't mix with current docs.

## Step 2 — For each current document, extract:
- Title: first # heading in the file
- Status: look for **Status**: line
- Date: look for **Date**: line
- One-line summary: second line after the title or ## Overview section

## Step 3 — Write docs/README.md

```markdown
# Project Documentation
> Auto-generated by /docs command — last updated {today}
> Run `/docs` in Claude Code to regenerate this index

---

## Architecture

| Document | Description | Updated |
|----------|-------------|---------|
| [SCRATCHPAD.md](architecture/SCRATCHPAD.md) | Codebase map, workflow graph, architecture diagram | {date} |

---

## Technical plans

| Document | Status | Date | Summary |
|----------|--------|------|---------|
{one row per TECH_DOC file found — most recent first}

---

## Archive
{timestamped copies listed here — collapsed, just filenames}

---
*Generated by engi — Claude Code Engineering Intelligence*
*Run `/docs` to refresh this index*
```

## Step 4 — Confirm
Tell the user:
- How many documents were indexed
- The docs/README.md has been updated
- "Run /docs any time the index gets out of sync"
